Introduction

What is git? According to git-scm website, is a free and open-source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

If I tell you more about other topics about git and its mechanism or features, you will be overwhelmed and afraid to continue, I will show you how to download and install git on your system instead.

I assumed you already downloaded the git, just open the installer and choose Next until the installation is finished. Then check if the installation exists by opening up Terminal (Linux and MacOS) or Command Prompt (Windows).

  • Windows: pressing Win+R button to open the "Run" window, then type cmd to open the Command Prompt window
  • MacOS: Applications > Utilites > Terminal
  • Linux: pressing the CTRL+ALT+T button to open the Terminal

Type git -v and press Enter on your keyboard to check the installed git version.

Setup

Using git can be done locally and remotely, initializing on a local environment is called a local repository, the repository is a whole git environment for your entire project that contains all your project files and directories.

To use git right away, you need to make a git environment on a folder/directory. Initialize it by using these commands on your preferred Terminal:

cd /path/to/project/directory
git init

Creating an empty repository will generate these files and is recognized by git:

  • .git directory, this file represents the local repository head file, by default this is hidden, if this file is modified or deleted, then this local repository is no longer recognized by git
  • .gitignore file, this is a file that excludes the file and directory listed inside here
  • README.MD file, this is a readme file that can be used on the GitHub repository landing page
Commands

To initialize the git repository until committing to the local repository (continuing from the Setup section), use these commands in order:

cd /path/to/project/directory
git init
git add .
git commit

Here is the rundown of commands:

  • Head to your project directory
  • Initialize an empty git repository
  • Between these commands, you can work on your project by adding and editing files to be committed later as your project checkpoint
  • Add all existing files (by using the "." sign) into the staging area
  • Commit your staged files into your local git repository
Environments

There are two environments of git, local and remote repository, local can be used for the local repository while remote is a hosted repository that can be accessed over the internet.

Here are the advantages of a local repository:

  • Can only be accessed locally such as LAN and above (except over the internet) so the repository is more private
  • Removes the need for hosting
  • Removes the need for an internet connection
  • Avoid network issues
  • Removes the need for backup files, acts as a save and checkpoint in case of a messed up project, you can go back to the previous checkpoint

Here are the advantages of a remote repository:

  • Easy cross-platform access to projects
  • Enabling collaboration
  • Project visibility

Let's assume you already made a git repository hosted on GitHub and copied the remote address like the above picture. Continuing from the last commit commands, here is the command for pushing a local commit into a newly created remote repository on GitHub as an example. Depending on your working branch name, in this case, it is on the "master" branch, so the commands are: 

git remote add origin https://github.com/my-username/repository-name.git
git push -u origin master

Every time you are done on local commits and want to push your commits into the GitHub remote repository you need to type this in Terminal: 

git push -u origin master
Branches

Need to experiment with something or add a feature to your project and don't want to mess up an existing project? git provides branching for your needs, branching is done locally by using this command and switching to your desired branch name: 

git checkout -b your-branch-name
git checkout your-branch-name

If you want to switch back to the main branch:

git checkout main

You might want to work on your project and make changes, then commit your changes on the local branch, in case if you forget how to commit you can go back to the Commands section. Then if you are done on the local branch and want to push into remote with that feature branch, you can do this on Terminal: 

git push -u origin your-branch-name-same-as-local

The code right before this is done one time only, the next time you want to push your committed changes from the local branch into the remote branch, you can do this instead: 

git push

Here are the benefits of branching:

  • Uninterrupted work and Commit action while collaborating with others
  • Does not interrupt the main branch of the project, usually when it is live
  • Easily track any changes and manage versions
Syncing

If you are working on a remote repository too and need to maintain your local in sync with the remote, you need two commands called git pull and push. But first, update your current local branch so you know if your local branch is already in sync or behind your remote branch, and check your status afterward. 

git remote update
git status

These are the definitions of git pull and git push, so you know what to do next.

  • Pull: an act of pulling updates from the remote into your local repository, this happens if your local is behind your remote, if you invoke this command and your local is already in sync, then the message shows "already up to date"
  • Push: an act of pushing your local changes into remote, will make a checkpoint of your local to remote, this happens every time you commit your changes on your local project

If your local branch commits are behind remote branch commits, do this command every time after the git remote update: 

git pull

If your local is ahead of remote (after the commit has been done), do this command every time you want to push your commit into remote:

git push

Now you can check your commit log by using this command (after that, quit by pressing q on your keyboard):

git log --oneline

Introduction

What is git? Refer from git-scm website, is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

If I tell you more about other topics about git and its mechanism or features, you will be overwhelmed and afraid to continue, I will show you how to download and install git on your system instead.

I assumed you already downloaded the git, just open the installer and choose Next until the installation finished. Then check the installation existed by opening up Terminal (Linux and MacOS) or Command Prompt (Windows).

  • Windows: pressing Win+R button to open "Run" window, then type cmd to open Command Prompt window
  • MacOS: Applications > Utilites > Terminal
  • Linux: pressing CTRL+ALT+T button to open Terminal

Type git -v and press Enter on your keyboard to check installed git version.

Setup

Using git can be done on local and remote, initializing on local environment is called local repository, repository is a whole git environment for your entire project that contains all your project files and directories.

To use git right away, you need to make git environment on a folder/directory. Initialize it by using these commands on your preferred Terminal:

cd /path/to/project/directory
git init

Creating empty repository will generate these files and is recognized by git:

  • .git directory, this file represents local repository head file, by default this is hidden, if this file is modified or deleted, then this local repository is no longer recognized by git
  • .gitignore file, this is a file that excludes file and directory listed inside here
  • README.MD file, this is a readme file that can be used on GitHub repository landing page
Commands

To initialize git repository until commiting to local repository (continuing from Setup section), use these commands in order:

cd /path/to/project/directory
git init
git add .
git commit

Here is the rundown of commands:

  • Head to your project directory
  • Initialize an empty git repository
  • Between this commands, you can work on your project by add and edit files to be commited later as your project save checkpoint
  • Add all (using "." sign) existing files into staging area (more on that later)
  • Commit your staged files into your local git repository
Environments

There are two environments of git, local and remote repository, local can be used for local repository while remote is a hosted repository than can be accessed over internet.

Here are the advantages of local repository:

  • Can only be accessed on local such as LAN and above connection (except over internet) so the repository is more private
  • Removes the need of hosting
  • Removes the need of internet connection
  • Avoid network issues
  • Removes the need of backup files, act as an save and checkpoints in case of messed up project you can go back to previous checkpoint

And here are the advantages of remote repository:

  • Easy cross platform access of projects
  • Enabling collaboration
  • Project visibility

Let's assume you already make git repository hosted on GitHub and copied the remote address like the above picture. Continuing from last commit commands, here is the command for pushing local commit into newly created remote repository on GitHub as example. Depending on your working branch name, in this case it is on "master" branch, so the commands are: 

git remote add origin https://github.com/my-username/repository-name.git
git push -u origin master

Everytime you are done on local commits and want to push your commits into GitHub remote repository you need to type this in Terminal: 

git push -u origin master
Branches

Need to experiment something or adding a feature on your project and don't want to mess up existing project? git provides branching for your needs, branching is done locally by using this command and switch to your desired branch name: 

git checkout -b your-branch-name
git checkout your-branch-name

If you want to switch back to main branch:

git checkout main

You might want to work on your project and make changes, then commit your changes on local branch, in case if you forget how to commit you can go back to Commands section. Then if you are done on local and want to push into remote with that feature branch, you can do this on Terminal: 

git push -u origin your-branch-name-same-as-local

The code right before this is done one time only, the next time you want to push your commited changes on local you can do this instead: 

git push

Here are the benefits of branching:

  • Uninterrupted works and commits while on collaboration with others
  • Does not interrupt the main branch of the project, usually when it is live
  • Easily track any changes and manages versions
Syncing

If you are working on remote repository too and need to maintain your local in sync with remote, you need two commands called git pull and push. But first update your current local branch so you know if your local branch is already in sync or behind from your remote branch, and check your status afterwards. 

git remote update
git status

These are the definition of git pull and git push, so you know what to do next.

  • Pull: an act of pulling updates from remote into your local repository, this happens if your local is behind your remote, if you invoke this command and your local is already in sync, then the message shows "already up to date"
  • Push: an act of pushing your local changes into remote, will make a checkpoint of your local to remote, this happens everytime you commit your changes on your local project

If your local branch commits are behind remote branch commits, do this command everytime after git remote update: 

git pull

If your local is ahead of remote (after commit have been done), do this command everytime you want to push your commit into remote:

git push

Now you can check your commit log by using this command (after that, quit by pressing q on your keyboard):

git log --oneline
